Swap elements
Flag Question

Problem Description

Given an integer array A of size N. In the array, you can swap (A[i],A[j]) if the following conditions are satisfied:

  1. i < j
  2. i is a divisor of j

You are also provided M independent queries. In each query, you are given two integers idx and K.

For each query, you are required to output the maximum number you can retrieve at index idx after performing at most K swaps.

NOTE 1: Queries are given by a 2-D integer array B of size M * 2 where B[i][0] denotes idx and B[i][1] denotes K.

NOTE 2: Consider array A as 1-based indexed.



Problem Constraints

1 <= N,M <= 50000
1 <= A[i] <= 109
1 <= B[i][0] <= N
0 <= B[i][1] <= 100



Input Format

First argument is an integer array A of size N.
Second argument is a 2-D integer array B of size M.



Output Format

Return an integer array of size M denoting output for each query as decribed above.



Example Input

Input 1:

 A = [2, 4, 6, 3, 2]
 B = [ 
       [5, 1]
       [4, 1]
       [4, 0] 
     ]

Input 2:

 A = [3, 4 9]
 B = [
       [2, 1]
       [1, 1]
     ]



Example Output

Output 1:

 [2, 4, 3]

Output 2:

 [4, 9]



Example Explanation

Explanation 1:

 For first query idx=5 and K=1. You can only swap element which is at index 1 (since 1 is the only divisor of 5 less than 5).
 For second query idx=4 and K=1. You can swap element which is at index 1 or 2 (since 1 and 2 are the divisor of 4 less than 4).
 For third query idx=4 and K=0. You cannot swap any element since K=0.

Explanation 2:

 For first query idx=2 and K=1. You can only swap element which is at index 1 (since 1 is the only divisor of 2 less than 2). max(4,3) = 4   
 For second query idx = 1 and K=1. You can swap element which is at index 3, So that value at index 1 is 9.

      Problem Description
      Given a permutation of size N containing all numbers from 1 to N. Find the number of permutation of size N which are less than or equal to B distance from the given permutation.
Distance between two permutations is defined as the number of indices where these two permutations are not equal.
      

      For Example: P1 = [1, 2, 3] and P2 = [2, 1, 3], Distance between these two permutation is 2.
      
because P1[0] != P2[0] and P1[1] != P2[1].
      

      NOTE:  Return the answer modulo 109 + 7 as the answer can be very large.
      

      

      

      

      

      

      

      

      

      

      

      

      

      

      



      Problem Constraints
      1 <= N <= 100000
      
1 <= A[i] <= N
      
0 <= B <= N


      Input Format
      First argument is an integer array A of size N.
      
Second argument is an integer B.


      Output Format
      Return an integer denoting the number of permutations which are less then or equal to B distance from the given permutation.
      



      Example Input
      Input 1:
      

       A = [1, 2, 3]
 B = 2
      

      Input 2:
      

       A = [4, 5, 1, 2, 3]
 B = 1
      

      

      

      

      

      

      

      

      

      

      

      

      

      



      Example Output
      Output 1:
      

       4
      

      Output 2:
      

       1
      

      

      

      

      

      

      

      

      

      

      

      

      

      



      Example Explanation
      Explanation 1:
      

       4 permutations exists with distance less than or equal to 2.
 Distance between [1, 2, 3] and [1, 2, 3] = 0.
 Distance between [1, 2, 3] and [1, 3, 2] = 2.
 Distance between [1, 2, 3] and [2, 1, 3] = 2.
 Distance between [1, 2, 3] and [3, 2, 1] = 2.
      

      Explanation 2:
      

       Only permutation with distance <=1 with given permutaion is only the permutation itself.
 Distance between [4, 5, 1, 2, 3] and [4, 5, 1, 2, 3] = 0.

        Problem Description
        Given a tree with N nodes numbered from 1 to N.
        

        Each node has a lowercase character associated with it.
        

        You are given Q queries, in each query you are given two integers u and v and you are required to find the character with a minimum non-zero frequency in a simple path from u to v.
        

        NOTE:
        

          

        • If there is more than one character with minimum frequency, then output the lexographically smallest character.
          • 

        • Each edge in the tree is bi-directional.
        

        

        

        

        

        

        

        

        

        



        Problem Constraints
        2 <= N, Q <= 105
        

        1 <= u, v <= N  and u != v
        

        

        

        



        Input Format
        First argument is an character array A of size N denoting character assigned to each nodes.
        

        Second argument is a 2-D array B of size (N-1) x 2 denoting the edge of the tree.
        

        Third argument is a 2-D array C of size Q x 2 denoting the queries, C[i][0] denotes u and C[i][1] denotes v.
        

        

        

        

        

        

        

        

        



        Output Format
        Return a string of size Q denoting answer for each query.


        Example Input
        Input 1:
         
        

         A = ['a', 'a', 'b']
 B = [ 
        [1, 2]
        [2, 3]
     ]
 C = [
        [1, 3]
        [2, 3]
     ]

        

        Input 2:
        

         A = ['a', 'b']
 B = [
        [1, 2]
     ]
 C = [
        [1, 2]
        [2, 1]
     ]

        

        

        

        

        

        

        

        

        



        Example Output
        Output 1:
        

         "ba"

        

        Output 2:
        

         "aa"

        

        

        

        

        

        

        

        

        



        Example Explanation
        Explanation 1:
        

         For Query 1: u = 1 and v = 3 so the path from 1 to 3 is 1 -> 2 -> 3 , 'a' occurs for two times on this path and 'b' occurs once.
 So 'b' will be the minimum frequency character in the path from 1 to 3.

 For Query 2: u = 2 and v = 3 so the path from 2 to 3 is 2 -> 3 , 'a' and 'b' each occurs once on this path.
 As both 'a' and 'b' are the minimum frequency character so we will output the lexographically smallest one i.e 'a'.

        

        Explanation 2:
        

         For Query 1: u = 1 and v = 2 so the path from 1 to 2 us 1 -> 2 , 'a' and 'b' occurs once on this path.
 As both 'a' and 'b' are the minimum frequency character so we will output the lexographically smallest one i.e 'a'.

 Query 2 is same as Query 1.

          Given a Tree consisting of A nodes. You need to perform depth first search traversal of a tree.
          

          

          

          There can be multiple depth first search traversal of a tree. Suppose C[i] represents the node visited after i-1 nodes in one of the such traversal.
          

          Goodness values of a traversal is sum of C[i] * i for all i from 1 to N. Find the minimum goodness value possible.
          

          Nodes are connected by N-1 edges. Given an array B of size N-1 where B[i](0-indexed) is the parent of the node (i+2).
          

          NOTE: 
          

          • Consider the above tree rooted at node 1
            • 

          • B[0] is the parent of node 2, B[1] is the parent of node 3 and so on.
            • 

          • As the answer can be large, return the answer modulo 109 + 7.
          

          

          

          

          

          

          



          Problem Constraints
           1 <= N <= 105
          
1 <= B[i] <= N


          Input Format
          First argument is an integer A representing the number of nodes in the tree.
          
Second argument is an integer array B of size N-1.



          Output Format
          Return an single integer denoting the minimum goodness value possible.


          Example Input
          Input 1:
          

          A = 5
B = [1, 1, 2, 2]
          

          Input 1:
          

          A = 5
B = [1, 1, 1, 2]
          

          

          

          

          

          

          

          

          

          

          

          



          Example Output
          Output 1:
          

           51
          

          Output 2:
          

           50
          

          

          

          

          

          

          

          

          

          

          

          



          Example Explanation
          Explanation 1:
          

          Given Tree:
           1
         /   \
        2     3
       / \
      4   5
 There are four different dfs order traversal of this tree.
 [1, 2, 4, 5, 3] and goodness value is 1*1 + 2*2 + 4*3 + 5*4 + 3*5 = 52.
 Second way is to traverse node 3 after node 1 i.e. [1, 3, 2, 4, 5] and goodness value is 1*1 + 3*2 + 2*3 + 4*4 + 5*5 = 54.     
 [1, 2, 5, 4, 3] and goodness value is 1*1 + 2*2 + 5*3 + 4*4 + 3*5 = 51
 [1, 3, 2, 5, 4] and goodness value is 1*1 + 3*3 + 2*2 + 5*4 + 4*5 = 54
 So, minimum goodness value possible is 51.

          

          Input 2:
          

          Given Tree:
            1
         /  |  \   
        2   3   4   
       /   
      5   
 Minimum goodness value occur for traversal [1, 4, 2, 5, 3] i.e. (1*1 + 4*2 + 2*3 + 4*5 + 5*3) = 50
